Definition of Done (DoD)

The Definition of Done (DoD) defines the criteria that must be met for any user story, task, or feature to be considered complete by the team. All work items must satisfy all of the following conditions before being marked as Done.

Code & Version Control

✔ Code is fully implemented and saved in a dedicated feature branch on GitHub.

✔ Code follows clean, readable, and consistent formatting standards with no errors or warnings.

✔ The feature is successfully merged into the main or develop branch without conflicts.

Testing & Validation

✔ The feature is manually tested and works as expected.

✔ Any applicable basic unit tests run successfully without failures.

✔ The feature runs correctly in the local or test (staging) environment.

Review & Collaboration

✔ Code has been reviewed or checked by at least one other team member (peer review).

✔ Feedback from the review (if any) has been addressed before merging.

Documentation

✔ The README or supporting documentation is updated with a brief explanation of how the feature works.

✔ Relevant supporting documents or screenshots are added to the /docs folder if required.

Task Completion

✔ The related Jira task is updated and marked as “Done.”

✔ Completion Rule

A feature or user story is considered Done only when all the above criteria are fully met
